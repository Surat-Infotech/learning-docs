# QA Testing Course — Manual QA from Zero to Job-Ready

Welcome! 👋 This is a complete, beginner-friendly course on **manual Quality
Assurance (QA) testing**. It takes you from "I've never tested software" all the way
to "I can run a full testing process, write professional bug reports, and walk into a
QA interview with confidence."

Every lesson is written in **simple, easy English**. You need **no coding
experience** and no prior testing knowledge. Just go in order, do the exercises, and
practice on real apps as you learn.

> **Want a schedule?** Follow the
> [6-Week Weekly Roadmap (10 hours/week)](WEEKLY-ROADMAP.md). It tells you *what* to
> study and practice *when*; this course teaches you the *how* in detail.

---

## Who This Course Is For

- Anyone wanting to **start a career in QA / software testing**.
- Developers who want to **test their own work** better.
- Career switchers looking for a **tech role that doesn't require heavy coding**.

This is a **manual QA** course. It builds the foundation every tester needs. If you
later want to move into **test automation**, the
[final module](05-tools-and-process/05-intro-to-automation.md) shows you the path
(and your repo's [JavaScript course](../javascript-course/) is a natural next step).

---

## How to Use This Course

1. **Go in order.** Each module builds on the one before it.
2. **Do the exercises.** Every lesson ends with them — they turn reading into skill.
3. **Practice on real apps.** Pick a website or app you use and test it as you learn.
   You'll find real bugs — that's the point.
4. **Keep a portfolio folder** from day one. Save your best test cases and bug
   reports; you'll need them for interviews
   ([Module 6](06-mastery/01-building-a-qa-portfolio.md)).
5. **Think, don't just click.** The best tool you own is the
   [tester's mindset](00-getting-started/02-the-testers-mindset.md).

**Suggested pace:** about 40% reading and 60% hands-on testing and exercises.

## What You'll Be Able to Do by the End

- Explain core QA concepts fluently (the stuff interviews ask about).
- Write clear **test scenarios** and **test cases** using proper design techniques.
- Find bugs systematically and write **bug reports** developers love.
- Classify bugs by **severity and priority** and track them through their life cycle.
- Produce **test plans** and a **traceability matrix**.
- Test across **browsers and devices**, and do basic **API testing** with Postman.
- Understand **Agile/Scrum** and where QA fits.
- Build a **portfolio** and prepare for **QA interviews**.

## A Note on Tools

professional tools (Jira, TestRail, Postman) are introduced in
[Module 5](05-tools-and-process/01-test-management-tools.md), but the **concepts**
come first — tools are easy once you understand the ideas.
